# 真实账号路径验收清单（录屏式）

> 路径：登录 → VPS → 文件 → 下载 → 审批  
> 目标：用真实账号按步骤点一遍，必要时录屏；本清单可勾选。  
> 配套脚本：`bash scripts/path-walkthrough-smoke.sh`（无登录态的网关/重定向/公开页检查）

## 0. 环境与前置

| # | 检查项 | 通过标准 | ☐ |
|---|--------|----------|---|
| 0.1 | 服务在线 | `systemctl is-active vcontrolhub-next` 为 active；`/login` 返回 200 | ☐ |
| 0.2 | 主题切换 | 登录前/后均可切 **浅色 / 深色**，无整页白块/黑块失控 | ☐ |
| 0.3 | 视口 | 至少测 **桌面 ≥1280** 与 **手机宽 ~390**（或 DevTools 模拟） | ☐ |
| 0.4 | 账号权限 | 账号至少具备：`server:read`（最好 `server:write`/`server:ssh`）、`storage:read`（最好 `storage:write`）、`command:read`（审批需 `command:approve`） | ☐ |

**建议录屏片段**

1. 登录 + 主题切换  
2. VPS 列表 → 详情 →（可选）SSH  
3. 文件浏览 → 上传/下载  
4. 下载中心创建任务  
5. 审批页处理一条待审请求

## 6. 跨路径串联（录屏重点）

按顺序操作，确认状态会互相反映：

1. **登录** → 仪表盘可见 VPS/队列摘要  
2. **VPS**：确认至少一台 enabled（或新建）  
3. **文件**：在绑定存储节点的路径下确认可列目录  
4. **下载**：创建指向该节点的下载任务 → 在下载页看到 RUNNING/完成  
5. **审批**：若触发命令审批（模板部署/助手/手动请求）→ 在 `/requests` 处理 → 回 VPS/任务页看结果  

| # | 串联检查 | 通过标准 | ☐ |
|---|----------|----------|---|
| 6.1 | 仪表盘数字 | 与各列表大致一致（允许短暂延迟） | ☐ |
| 6.2 | 通知铃铛 | 审批/下载类事件有通知（若开启） | ☐ |
| 6.3 | 深浅色全路径 | 上述页面切换主题后无对比度崩坏 | ☐ |
| 6.4 | 手机全路径 | 底栏 5 入口 + 设置可达；无横向整页滚动 | ☐ |

## 9. 权限速查

| 页面 | 关键权限 |
|------|----------|
| `/servers` | `server:read`；写：`server:write`；终端：`server:ssh` |
| `/files` | 会话即可进页；写：`storage:write` 等 |
| `/downloads` | 读：`storage:read`；写：`storage:write`；节点：`storage:manage-node` |
| `/requests` | 读列表需登录；审批：`command:approve` |
